# v1.4.0 — Base-address-aware TGOS data

This release publishes data kit version `2026.07.23.1` with address policy
version `2`. SQLite schema versions are unchanged, so existing schema-v3
consumers remain compatible.

## Highlights

- Consolidates floor suffixes by exact coordinate plus complete base address,
  rather than deleting every address that shares a coordinate.
- Preserves distinct house numbers assigned to the same TGOS building-centre
  point.
- Prefers an existing base-address row; when a group contains only floor
  rows, rewrites the lowest-id row to the searchable base address.
- Treats `樓` or `層` as a floor suffix only when it occurs after `號`, so
  proper names such as `合作大樓１之１號` remain unchanged.
- Adds release-facing `data_version` and `address_policy_version` metadata,
  manifest fields, ZIP sidecars, and automated release preflight checks.

record should apply a deterministic tie-break for different addresses at the
same distance.
